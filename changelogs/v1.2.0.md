# Magazzino v1.2.0 — 2026-06-10

The dashboard becomes a real control center: views, live status dots,
tabbed preview — plus project-wide search, tmux sessions and parallel pull.

## Added
- **Views in the dashboard**: `Tab` cycles *all → pinned → dirty → recent →
  archive*. The border label and prompt show the active view; empty views are
  skipped. In the *archive* view `ctrl-a` restores instead of archiving.
- **Status dots in the list**: every project shows a dot — yellow (dirty),
  magenta (unpushed), green (clean), dim (not a repo). Powered by a cache in
  `~/.local/share/magazzino/status-cache`, refreshed in the background with a
  60s TTL so rendering stays instant. Force it with `mag refresh`.
- **Tabbed preview**: `ctrl-o` cycles *info → readme → todo → github* in the
  preview pane (dashboard and picker). README renders with `glow` or `bat`
  when available; *todo* greps TODO/FIXME/HACK; *github* shows open PRs and
  recent CI runs via `gh`.
- **`mag grep <query>`** (`g`): full-text search across every root (ripgrep
  when available), results in fzf with context preview, enter opens your
  editor at the exact file:line (`code -g`, `nvim +line`, …).
- **`mag t [name]`** (`tmux`): one tmux session per project — created on
  demand, attached outside tmux, switched inside it. Also bound to `ctrl-t`
  in the dashboard.

## Changed
- **`mag pull` is parallel** (4 workers): results stream in as repos finish,
  same per-repo report and summary; the status cache refreshes right after.
- Dashboard legend updated with the new keys (`⇥ view`, `^t tmux`,
  `^o preview`).
